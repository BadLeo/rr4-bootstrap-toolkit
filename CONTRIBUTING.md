# Contributing

## Scope

This repository contains a static RR4-inspired UI showcase and cookbook. Keep contributions aligned with that direction: focused, visually coherent, and easy to review.

## Development Workflow

1. Fork the repository or create a feature branch from `main`.
2. Keep changes small and scoped to one concern.
3. Test changes locally in a browser before opening a pull request.
4. Update documentation when behavior, structure, or usage changes.

## Coding Guidelines

- Preserve the existing visual language unless the change intentionally evolves it.
- Prefer semantic HTML and accessible attributes.
- Keep JavaScript framework-free unless there is a clear reason to change the stack.
- Avoid unnecessary dependencies or build tooling for this static site.
- Respect reduced-motion behavior when adding new animations.

## Pull Requests

Please include:

- a concise summary of the change
- screenshots or a short screen recording for UI changes
- testing notes describing what you verified
- linked issues when relevant

## Issues

Use GitHub Issues for bugs, feature requests, and documentation gaps. Include reproduction steps and browser details for rendering bugs.
