# Security Policy

## Supported Versions

Security updates are applied to the latest version on the default branch.

## Reporting A Vulnerability

Please do not open public GitHub issues for security-sensitive reports.

Instead, report vulnerabilities privately to the repository maintainer with:

- a clear description of the issue
- affected files or pages
- reproduction steps or proof of concept
- impact assessment if known

You should expect an acknowledgement within a reasonable time and remediation based on severity and exploitability.
